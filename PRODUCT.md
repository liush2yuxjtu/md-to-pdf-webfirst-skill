# Product

## Register

brand

## Users

Codex agents and users who need Markdown or report HTML turned into polished, reader-ready PDFs.

## Product Purpose

This repository packages the `md-to-pdf-webfirst` skill. It helps agents build a print-friendly HTML surface first, apply report design craft, print with Chrome, and verify the resulting PDF with previews and metadata.

## Brand Personality

Editorial, exacting, evidence-led.

## Anti-references

Raw browser printouts, tiny table typography, generic dashboard cards, decorative consulting skins, and report pages that look mechanically exported.

## Design Principles

- HTML is the design surface; PDF is the final render.
- Readability beats page-count compression.
- Each exhibit should state one judgment before showing evidence.
- Publication-mode reports need imagery, figures, front matter, footnotes, and a stable page system.
- Generated assets are acceptable only when they clarify the report and survive print review.

## Accessibility & Inclusion

Target readable A4 print output with strong text contrast, stable hierarchy, non-color-only status cues, and sufficient type size for dense Chinese tables.
