最终流程

直接把 Markdown 转 PDF 往往像纯文本导出。这个 skill 把 HTML 当成 PDF 的设计画布，先解决封面、目录、章节、代码块、分页，再打印。

01

读取本地 `.md` 或远程 Markdown URL，保留源文件与哈希。

02

生成 PDF-friendly HTML：封面、section map、章节页、代码块、A4 CSS。

03

用 headless Chrome 打印 PDF，并关闭默认页眉页脚。

04

用 `pypdf` 验证页数和可抽取文本，再生成真实 PDF 首页预览。

安装和运行

把 skill 目录放进 Codex skills folder 后，脚本就能复现这个流程。

git clone https://github.com/liush2yuxjtu/md-to-pdf-webfirst-skill.git
cp -R md-to-pdf-webfirst-skill/md-to-pdf-webfirst ~/.codex/skills/md-to-pdf-webfirst

python ~/.codex/skills/md-to-pdf-webfirst/scripts/md_to_pdf_webfirst.py \
  --input https://code.claude.com/docs/zh-CN/best-practices.md \
  --slug zh-cn-best-practices-webfirst \
  --out-dir outputs
